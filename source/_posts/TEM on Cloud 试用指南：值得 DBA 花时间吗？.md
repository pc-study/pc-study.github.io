# 前言

在数字化转型的浪潮中，TiDB 作为分布式数据库的佼佼者，正被越来越多的企业采用。然而，随着集群规模的扩大和业务复杂度的提升，运维管理成为了摆在 TiDBer 面前的一道难题：多集群切换繁琐、命令行操作复杂、故障排查效率低下...

为了彻底解决这些痛点，TiDB 数据库运维管理平台（TEM）应运而生。这款一站式管理平台将复杂的运维操作简化为可视化的白屏操作，让企业级 TiDB 集群管理变得前所未有的简单。

> TEM 试用报名进群获取链接：**https://forms.pingcap.com/f/tem-trial-use-event**

**7 月 25 日 - 9 月 1 日**，TEM on Cloud 试用活动正式启动！无论你是已经在云上部署了多套 TiDB 的资深用户，还是正在调研云端数据库解决方案的新手，这都是一个绝佳的体验机会。参与试用并提供反馈，不仅能获得丰富的 TiDB 周边礼品，还有机会获得 PCTA/PCTP/PCSD 考试兑换券等超值奖励。

# TEM 产品介绍

## 核心价值

平凯数据库企业级运维管理平台（TEM）是专为 TiDB 打造的全生命周期管理平台，支持平凯数据库 v7.1 及 TiDB 社区版 v6.5 以上版本。通过统一的 Web 界面，用户可以轻松完成：

**核心功能模块：**

- 🚀 **集群管理**：部署/纳管/升级一键完成；
- ⚙️ **配置优化**：参数配置智能推荐；
- 📈 **弹性扩容**：节点扩缩容秒级响应；
- 📊 **全面监控**：实时监控告警，故障预警；
- 💾 **备份策略**：自动化备份，数据安全无忧；
- 🔧 **智能诊断**：故障自愈与性能优化；
- 📅 **任务调度**：定时任务自动执行；
- 📈 **资源可视**：CPU、内存、磁盘 I/O 实时监控；

![](https://files.mdnice.com/user/16270/553d2a44-17a6-44ef-bf8d-0e4259de2384.png)

## DBA 工作流程的重构

传统 DBA 工作流程在 TEM 平台下的变化：

**变更管理流程**

```
传统流程：需求分析 → 方案设计 → 手动执行 → 结果验证
TEM 流程：需求分析 → 平台配置 → 自动执行 → 结果确认
```

**故障处理流程**

```
传统流程：告警接收 → 手动排查 → 问题定位 → 手动恢复
TEM 流程：智能告警 → 自动诊断 → 问题预测 → 自动修复
```

**容量规划流程**

```
传统流程：资源监控 → 趋势分析 → 容量评估 → 扩容执行
TEM 流程：智能监控 → 自动预测 → 弹性扩容 → 成本优化
```

# TEM 快速部署指南

## 部署准备

官方提供了详细的部署教程：

> TEM on 腾讯云安装 + 试用指南：https://tidb.net/blog/3cb0dabf

## 关键配置步骤

**1. 获取试用资格**

- TEM 试用报名进群获取链接：**https://forms.pingcap.com/f/tem-trial-use-event**

**2. 服务器配置建议**

- **推荐区域**：广州七区（资源充足）；
- **网络配置**：开启公网 IP，配置安全组放行 32000 端口；
- **主机配置**：中控机和集群主机使用内网 IP；

**3. 集群部署规范**

- **组件配置**：推荐 `2C2G` 配置；
- **最小部署**：`1PD + 1TiDB + 1TiKV + 1Prometheus + 1Grafana`；
- **版本要求**：TiDB v6.5+ 且通过 TiUP 管理；

**4. 安全组配置**

- 确保 32000 端口在安全组中放行，以便访问 TEM 管理界面。

# 使用体验与总结

## 产品亮点

经过实际体验，TEM 展现出了以下优势：

**用户体验**

- 界面设计简洁直观，操作流程丝滑流畅；
- 从命令行到可视化的完美转变；
- 学习成本低，上手即用；

**功能完整性**

- 涵盖 TiDB 运维全生命周期；
- 监控告警体系完善；
- 自动化程度高，减少人工干预；

- 支持多集群统一管理；
- 故障自愈能力强；

# 写在最后

作为一个 DBA，我认为 TEM 代表了数据库运维管理的发展方向。它不是简单的工具升级，而是运维理念和工作模式的根本性变革。

![](https://files.mdnice.com/user/16270/55f2a971-77eb-4c0a-9420-6c6ed8dbf7b4.png)

**核心价值判断**

1. **技术先进性**：基于云原生理念的架构设计，符合技术发展趋势；
2. **实用性**：解决了分布式数据库运维的实际痛点，提升了工作效率；
3. **可扩展性**：平台化架构为未来功能扩展提供了良好基础；

**实施建议**

1. **渐进式采用**：从非核心环境开始，逐步扩大使用范围；
2. **团队培训**：投资团队技能升级，适应新的工作模式；
3. **流程重构**：基于平台能力重新设计运维流程和规范；

随着 AI 和机器学习技术的发展，我预期 TEM 这类平台将会集成更多智能化功能：

- 基于历史数据的容量预测和自动扩容；
- 智能 SQL 优化建议和自动索引推荐；
- 故障预测和自动修复能力；
- 成本优化和资源调度算法；

对于正在考虑技术升级的 DBA 团队，建议积极参与 TEM 试用活动，这不仅是对新技术的评估，更是对未来运维模式的探索和准备。
