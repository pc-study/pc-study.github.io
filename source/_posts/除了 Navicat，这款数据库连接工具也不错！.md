# 前言

嘿，各位同行！作为一个每天跟各种数据库打交道的 DBA，选个称手的数据库连接工具真是一个头疼的事。以前总被朋友问：“**有没有好用又免费的？**” 我一般得按数据库分着推荐——Oracle 用 PL/SQL Developer 或 SQL Developer，MySQL、PG 这些就用 Navicat Premium 或者 DBeaver。

不过现在，我基本上就统一安利 **SQLark** 了。这工具用起来是真不错，功能多还免费，关键官方特别听劝——**咱们提的需求他们真做，报的 Bug 也很快修复**，更新节奏稳得很！

这不，刚出的 V3.8 版本（**2025.12 发布**），又带来一波咱们 DBA 和开发喜闻乐见的升级。我赶紧试了试，挑几个我觉得特别实用的功能跟大家唠唠。

## 🛡️ 生产环境保护加强：防手滑神器

首先必须说这个！作为 DBA，最怕的就是不小心在生产环境动错了东西。SQLark 之前就能把连接标记成“**生产环境**”，这次 V3.8 在防护上又加了一道保险。

现在，只要你标记为生产库，进行下面这几类高风险操作时，都会**强制弹窗二次确认**：

- **改数据**：在表数据页或结果集中，编辑或删除数据，提交前会提示数据变更状态。
- **删对象/改结构**：无论是删表、删视图，还是改字段类型、索引，弹窗里都会**直接预览 DDL 语句**，让你看清楚再执行。
- **执行风险 SQL**：在查询窗口里跑 `DROP`、`TRUNCATE`、`DELETE` 或者 `ALTER` 这类语句，或者执行外部脚本时，**风险语句会高亮显示**，必须再点一次确认。
- **其他限制**：生产环境直接禁止开“自动提交”模式，也禁止连接 `localhost` 或 `127.0.0.1`，从源头降低误操作可能。

标记后，整个窗口都会有**醒目水印和彩色边框**时刻提醒你：“**这是生产库！手别抖！**”

![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008359935889989632_395407.png)

这功能对我们来说太踏实了，特别是忙晕了或者同时开好几个环境的时候。要是以后能自定义环境标签（比如“测试”、“报表”、“容灾”）和颜色，那就更完美了！

## 🔍 对象搜索：又快又准，告别卡顿

以前数据库一多，对象上万的时候，全局搜个东西总感觉有点卡。V3.8 把这部分彻底优化了，现在搜索流畅得很。

![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008375632590364672_395407.gif)

它的“查找对象”不仅能搜名字，还能**直接搜对象里的代码内容**（比如存储过程里某个关键字），特别适合找逻辑。现在按 `Shift+Shift` 或 `Ctrl+Shift+F` 就能快速调出搜索框。

最贴心的是，新增了**限定范围搜索**。以前是全局搜，结果里混着各个库的同名对象。现在可以指定在某个**模式**（Oracle/达梦/PG）或者某个**数据库**（MySQL）里搜，结果干净多了。

> 小技巧：点模式名后面的按钮，或者在搜索框直接输入“/模式名”就行。

![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008377240636694528_395407.gif)

开发同事经常要找某个表或函数，这个优化对他们来说简直是福音。

## 🔄 导入三方连接：一键搬家，省时省力

之前 SQLark 只能导入导出自己的连接配置。V3.8 终于支持**导入其他工具的连接信息**了，比如 DM Manager、DBeaver 这些。
- **支持导入的数据库类型**：达梦、Oracle、MySQL、PostgreSQL；
- **支持导入的连接信息**：连接名、主机、端口、用户名、URL 信息、SSH 连接信息等；
- **支持的文件格式**：`XML/JSON/NCX`；

![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008377849108062208_395407.png)

![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008378019870760960_395407.png)

这对想从其他工具迁移过来的朋友太友好了，不用一个个重新配。希望后续能支持更多客户端。

## 📂 数据导入：拖拽就完事，还能智能建表

数据导入功能也更“聪明”了。现在直接把 `xlsx`、`csv` 这些文件**拖拽到左侧导航树**就能触发导入。

- 如果**精准拖到某张表上**，就直接往这张表里导。
- 如果**拖到文件夹或空白区域**，它会先找有没有和文件同名的表；如果没有，居然能**直接根据文件内容，智能推荐表结构并创建新表**，你确认一下字段类型就能连表带数据一把导入了。

![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008378503037804544_395407.gif)
![](https://oss-emcsprod-public.modb.pro/image/editor/20260106-2008379147622637568_395407.png)

另外，现在也支持**只导入文件的前 N 行**，做测试和验证的时候特别方便。
